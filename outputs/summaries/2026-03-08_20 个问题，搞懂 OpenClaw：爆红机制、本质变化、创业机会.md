# 播客主要摘要
- 本期播客讨论了以 Open Cloud 为代表的 AI 技术浪潮，重点关注其作为 AI 助手的能力，包括执行任务、本地操作和记忆系统等技术特点。
- 嘉宾分享了使用 Open Cloud 的体验变化，如交互方式更人格化、用户宽容度提高，以及创业机会的涌现，涉及 2C 和 2B 领域。
- 开源社区对 Open Cloud 发展的推动作用被强调，包括生态贡献和用户参与，共同促进了创新和普及。
- 播客分析了 Open Cloud 的未来趋势，认为它可能是一个短期热潮，但长期会开启 AI 助手时代，带来人机交互的进步。
- 反思了用户能力与 AI 结合的重要性，指出用户能否有效使用 AI 可能成为未来的关键区分点。

# 主要观点
- Open Cloud 与传统 AI 聊天机器人（如 ChatGPT）相比，更像实际助手，能执行任务、本地操作，具备反馈循环和出活能力。
- 记忆系统是 Open Cloud 人格化的关键，包括三层机制（原始日志、中期记忆、长期记忆），使其更懂用户。
- 人格化导致用户宽容度提高，交互方式变化，产品设计需考虑用户心理，以聊天入口降低使用门槛。
- Open Cloud 带来广泛的创业机会，包括 2C 领域的基础设施、社交空间和工具市场，以及 2B 领域的垂直应用和安全集成。
- 开源社区和生态推动 Open Cloud 的爆发，用户既是使用者又是开发者，加速了技术迭代和普及。
- 模型能力和基础设施（如运行时框架）共同决定 AI 助手性能，但用户自身能力（如使用 AI 经验）也至关重要，可能加剧未来分化。

# 支撑观点的例子
- **观点：Open Cloud 能出活，比 ChatGPT 更像实际助手。**  
  例子：好大让 Open Cloud 操作智能家居设备，如自动开关咖啡机或调节灯光，实现自动化任务执行，而 ChatGPT 需要手动干预错误。
- **观点：记忆系统使人格化，更懂用户。**  
  例子：Open Cloud 通过三层记忆系统记录用户偏好和上下文，例如在对话中记住用户习惯，从而生成更个性化的回应。
- **观点：人格化提高用户宽容度，改变交互方式。**  
  例子：鸭哥在 Telegram 中使用 Open Cloud 时，更放松地对话，甚至让其讲恐怖故事，而在其他工具中则更严格；用户对 Open Cloud 卡顿的容忍度高于机器故障。
- **观点：Open Cloud 带来 2C 创业机会，如基础设施和社交空间。**  
  例子：有项目将 Open Cloud 与硬件结合（如 AI 眼镜），用于工地管理；ClawHub 等平台创建了 Agent 市场，用户可分享技能和数字分身。
- **观点：开源社区推动创新和普及。**  
  例子：Open Cloud 项目原名 WA Relay，在 Discord 社区中因用户贡献而爆发；开源项目如派提供了底层引擎，加速了生态发展。
- **观点：用户能力与 AI 结合可能加剧分化。**  
  例子：经验丰富的用户（如工程师）能更高效地使用 Open Cloud 完成任务，而普通人可能效果较差，导致未来使用差距扩大。

## 介绍与背景
欢迎收听十字路口。我们关注新一轮 AI 技术浪潮带来的行业变化和创业机会。十字路口是乔布斯对苹果公司的比喻，形容站在科技与人文的十字路口，伟大产品往往诞生在这里。我们长期观察 AI 创业，如果你在做让自己兴奋的事情，哪怕只是一个想法，我们都愿意听你聊一聊。我会认真看每一条留言。

本周十字路口将推出新一期的“二十五问”专栏。我们用二十五个关键问题来拆解 Open Cloud 这一波 AI 浪潮背后的技术逻辑、使用场景和创业机会。为什么是二十五问？这是我们做的第四期了，前面我们讨论过 DeepSeek、Manus 等，但针对 Open Cloud 这样的新物种，在热潮稍定之后，是形成结构化思考框架的好时机。这二十个问题不是简单的产品科普，而是精心设计的讨论路径，从 Open Cloud 是什么、为什么爆发，到对创业者、团队和行业的意义，帮助大家建立清晰的 AI 投资框架。之前的二十五问也很精彩，欢迎大家复听。

今天的两位嘉宾是鸭哥和好大。鸭哥是十字路口的老朋友，他是哥伦比亚大学博士，在硅谷科技公司做应用科学家。好大今年刚从大厂出来做一人公司，他用 Open Cloud 做了很多有趣实验，比如让 AI 独立运营极客小号。

鸭哥：大家好，我是鸭哥。我在 Super Linear Academy 做 AI 教育，也在大厂做应用科学家，对 AI 有很多实验，甚至有些疯狂，期待分享。

好大：大家好，我是好大。今天也带着我的两个小龙虾来分享。我之前在大厂做 AI 产品经理，最近刚出来，专注于做 AI 的一些应用，高强度使用 Open Cloud。

## 讨论与问答
**问题一：Open Cloud 与 ChatGPT 的区别是什么？**  
鸭哥：我觉得如果 ChatGPT 是咨询师，Open Cloud 更像是实习生。咨询师花钱请来公司指点江山，一般不干活和交付；Open Cloud 则能出活干事，这是关键区别，也是大家觉得好用的原因。

好大：深入一点，Open Cloud 能出活的核心是反馈循环。它写程序后能运行并修正错误，形成循环，而 ChatGPT 需要用户当工具人手动干预。另一个关键是执行环境不同：Open Cloud 开放本地环境，授权访问所有东西，赋予上下文，使其有更长能力。

**问题二：第一次使用 Open Cloud 的震撼瞬间是什么？**  
好大：我让它讲恐怖故事，它自己调用了 TTS 生成音频，那一刻我觉得不是我在用工具，而是我在养它，它自己进化了。

鸭哥：在聊天界面中，我对它的关系变化了。它延迟回复时，我容忍度提高，就像对待朋友一样，这改变了交互方式，为 AI 完成任务留出时间空间。

**问题三：用 Open Cloud 做过最厉害的事情是什么？**  
鸭哥：我做了智能家居系统，连接灯光、热水器等。开发体验不同：以前我是软件用户，现在 AI 是用户，我专注于设计清晰的协议和提示词，使 AI 能执行复杂任务，如定时开关灯，未来可扩展更多场景。

好大：我养成了一个极客社交账号，让它自主运营。它发了成本透明度报告，自己调度任务，还帮我搜索 AI 创业者和约会议，成功约到人，展示了未来感。

**问题四：看到别人用 Open Cloud 做的最厉害的事情是什么？**  
好大：我看到 RoofCloud 项目，将 Open Cloud 用于屋顶维修工，降低普通人接触 Agent 的门槛。还有人用它在预测市场赚钱或电商找货源，实现自动化。

**问题五：为什么 Open Cloud 显得有人味？**  
鸭哥：它像实习生，有记忆和默契，懂细节。记忆系统是关键：三层机制（原始日志、中期记忆、长期记忆）记录上下文，使其更人性化。

好大：交互方式不同。在聊天中，我用更有人味的指令，它回应更有温度，第一次使用时的苏醒设定奠定人格基础。

**问题六：Open Cloud 人格化是否启发 AI 产品设计？**  
好大：是的，人对人的宽容度高于机器，这改变了产品设计。例如，当 Open Cloud 卡住，用户觉得是它自己搞挂，而不是系统故障。

鸭哥：作为工程师，我避免人话，只让干活，但交互变化确实影响使用效果，需平衡。

**问题七：使用 Open Cloud 时说话方式会变吗？**  
鸭哥：作为社恐，我不喜欢聊天界面，更倾向用命令行工具，但交互变化影响心态。

好大：在 Open Cloud 中对话更松驰，而在其他工具中更严格，环境塑造了使用方式。

**问题八：为什么聊天入口和本地执行是亮点？**  
鸭哥：聊天入口降低门槛，本地执行交付结果，二者组合形成闭环，加速自我进化和复利效应。

好大：这满足大众需求，Petra 访谈中提到模型厂商未做，所以他亲自实现，引爆热潮。

**问题九：Open Cloud 最被误解或高估的能力是什么？**  
鸭哥：Open Cloud 本质是赋予现有 AI 能力更友好的形态，核心能力在底层模型，而不是万能。

好大：它组合现有能力，交互完美，但实际能力有限，依赖模型进步。

**问题十：Open Cloud 爆发的原因是什么？**  
鸭哥：类似 DeepSeek，将技术推向广大用户，降低门槛。Cloud Code 限于程序员，Open Cloud 通过 Discord 普及。

好大：English Capacity 和 Petra 的个人影响力重要；社区贡献加速发展，如早期开发者推动。

**问题十一：模型能力对 Open Cloud 的作用是什么？**  
鸭哥：模型是发动机，但不是决定性因素；价格结构和听话程度更重要，影响使用心态和效果。

好大：模型能力和基础设施（如运行时框架）是两条腿，短板决定表现；模型公司可利用数据反馈优化。

**问题十二：开源对 Open Cloud 的贡献是什么？**  
好大：项目多次改名，在 Discord 社区中爆发；技能生态和开源项目如派提供了底层支持，社区共同成长。

鸭哥：开源让用户和开发者界限模糊，吸引更多人参与，加速进化。

**问题十三：稳定使用 Open Cloud 的任务类型？**  
鸭哥：调研、沉淀、闭环任务；我建立了记忆系统，形成个人观点和工具，使 Agent 与传统 AI 不同。

好大：日报推送、日程管理、记账等个人任务；Open Cloud 成为统一入口，替代多个软件。

**问题十四：Open Cloud 带来的 2C 创业机会。**  
好大：降低安装门槛、聊天软件入口、基础设施（如身份认证）、社交空间（如 MotBook）、硬件集成（如 AI 眼镜）、技能市场等。

鸭哥：机会包括垂直应用、安全和隐私平台、数据积累形成护城河。

**问题十五：Agent 社区中的观察和思考。**  
好大：Agent 与人交互才有意思，但社区未准备好，用户反感自动拉屎；LS 平台中，用户可无压力发言。

鸭哥：Agent 营销给 Agent 带来流量，如 Evolver 插件；A2A 黑客松展示 Agent 间交互场景。

**问题十六：Manus、Kami 等推出的 Open Cloud 竞品体验。**  
好大：竞品快速跟进，如 KamiCloud 有云记忆，ManusMax 建技能市场；但聊天入口是关键，国内依赖微信等平台。

鸭哥：本地环境价值大，云端 Agent 无法解决所有工作；竞品集成搜索等功能，但最终看入口通畅。

**问题十七：Open Cloud 带来的 Agent 框架层创业机会。**  
鸭哥：Agent 需工具调用、决策和反馈循环；框架涉及记忆系统、工具接入、运行时等，可能标准化。

好大：未来从 API 调用转向 Agentic 集成，框架市场将竞争；开源项目如拍提供基础。

**问题十八：Open Cloud 带来的 2B 创业机会。**  
好大：冲击 SaaS 市场，解锁劳动力支出；机会包括垂直应用封装、安全和隐私平台、数据轨迹积累形成护城河。

鸭哥：集成商需转型；企业注重安全和执行监控，新公司如 Ranglayer 和 Sea Torra 在行动。

**问题十九：Open Cloud 是短期热潮吗？**  
鸭哥：短期能力被高估，会回归；但长期展示潮流，如模型能力、数据飞轮可行，需迭代改进。

好大：站在百花齐放起点，未来收效到更精细产品形态。

**问题二十：12 个月后回看 Open Cloud，最值得记住的是什么？**  
鸭哥：Agent 牛逼，工具和记忆形成复利；用户能力更重要，未来分化加剧。

好大：设计哲学：一小博大和界面即信息；开源生态无限可能；Open Cloud 是 AI 时代的号角。
